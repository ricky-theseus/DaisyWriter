# 技术博文指南 💻

写、批量、自动发布技术文章。

---

## 概览

```
拆解 (参考) → 写作 (起草) → 批量 (生产) → CSDN
                                              │
                                      sync-csdn / csdn-upload
```

---

## 技能 1：写作 (`tech-write`)

结构化技术博文写作。

```
/tech-write "用 FastAPI 构建 REST API"
```

**标准结构：**
1. **问题** — 解决什么问题？
2. **方案** — 分步实现
3. **代码** — 可复制运行的片段
4. **验证** — 如何测试
5. **总结** — 关键要点

**审查清单：**
- 所有代码片段可复制运行
- 版本号和 API 引用明确
- 逻辑链完整（无跳步）
- 无空白占位符
- 字数匹配目标读者

**输出：** `博文/{一级}/{二级}/{三级}/{文章}.md`

---

## 技能 2：拆解 (`tech-deconstruct`)

分析参考文章的结构模式。

```
/tech-deconstruct "参考文章路径"
```

**分析维度：**
- 结构骨架
- 开头钩子技巧
- 代码质量和示例
- 信息密度
- 逻辑链完整性
- 读者匹配度
- 可借用模式

---

## 技能 3：批量 (`tech-batch`)

批量生产多篇文章。

```
/tech-batch 项目目录
```

**双层循环架构：**
- **外层循环**：按计划调度条目
- **内层循环**：生产 → 审查 → 修改 → 再审（直到零问题）

**规则：**
- 生产者和审查者始终是独立 agent
- 审查者始终盲审（无对话历史）
- 无重试限制——迭代直到零阻断

---

## 技能 4：同步 CSDN (`sync-csdn`)

同步 CSDN 已发表文章到本地。

```
/sync-csdn
```

**流程：**
1. 获取 CSDN 主页
2. 与本地 `published.json` 对比
3. 将匹配的草稿移到 `已发表/`
4. 更新记录

---

## 技能 5：上传 CSDN (`csdn-upload`)

浏览器自动化上传草稿到 CSDN。

```
/csdn-upload --dry-run      # 预览三态分布
/csdn-upload                # 上传所有未发布草稿
/csdn-upload --login        # 首次登录
/csdn-upload --sync         # 同步发布状态
```

**三态管理：**

| 状态 | 含义 |
|------|------|
| 📝 **未上传** | 本地草稿，未上传到 CSDN |
| 📤 **草稿箱** | 已在 CSDN 草稿箱 |
| ✅ **已发布** | CSDN 上已公开 |

**安全：** 只存草稿，不会自动发布。

# 公众号文章写作 Skill

## 核心工作流

### 执行模式判断

IF 用户提供了现有草稿（粘贴了部分内容 / 说「帮我把这篇写完」「补全」「继续写」）：
  → 进入「补全模式」
  → 理解现有草稿的文章类型、核心主题、已有结构
  → 识别草稿缺失的部分（未完成的章节、缺少的结语等）
  → 以与现有内容一致的风格和语气补全缺失部分
  → 配图/审稿/发布流程正常走

ELSE（从零开始写新文章）：
  → 执行正常流程

### Step 1：确认文章类型

询问用户文章属于哪种类型：

| 类型 | 结构特征 |
|---|---|
| **科教/技术类** | 摘要+字数+章节+引用 |
| **活动预告类** | 封面+摘要+日程+报名方式 |
| **活动回顾类** | 封面+综述+图片+总结 |
| **动态/公告类** | 封面+摘要+正文 |

### Step 2：套用结构模板

**科教/技术类模板：**
```
# [标题]

内容摘要

1. 关键点A
2. 关键点B
3. 关键点C

**（全文约X字）**

01

[章节一标题]

[正文]

02

[章节二标题]

[正文]
```

**活动预告类模板：**
```
# [标题]

[封面图]

内容摘要

01

活动亮点

02

活动详情

📅 时间：
🏫 地点：
📡 形式：

03

报名方式
```

### Step 3：写正文

- **读者定位**：关心技术干货的读者，有一定基础但不是专家
- **字数目标**：技术类 2000-4000 字；预告/回顾类 800-2000 字
- **每张图下方写一行说明文字**

### Step 4：生成配图

为文章配图，注意风格统一。配图生成后建立配图索引。

### Step 5：转换为微信 HTML

**关键样式：**
- body: `font-size: 17px; line-height: 1.9; color: #333; max-width: 677px`
- H2: `color: #2d6a4f; border-left: 4px solid #2d6a4f; padding-left: 12px`
- strong: `font-weight: bold; color: #1a1a1a`
- 图片: `base64 嵌入，max-width: 100%; border-radius: 8px`
- hr: `border-top: 1px dashed #ccc`
- blockquote: `background: #f9f9f9; border-left: 4px solid #2d6a4f`

**修改方式**：文字/样式改动直接用 StrReplace 修改 HTML 文件。

### Step 6：添加底部模板

底部模板包含：署名 → 分隔线 → 版权声明 → 二维码（如有）

署名格式：
```
出品 | [机构名称]
文字 | [作者]
美编 | [编辑]
```

### Step 7：审稿

写完后审查：
- AI腔（元评论、防守修饰、套话）
- 标题是结论还是话题描述
- 绝对表达是否需软化
- 结语是否涵盖全文

### Step 8：发布指引

HTML 完成后告知用户发布步骤：
1. 用浏览器打开生成的 HTML 文件
2. 全选（Ctrl+A）→ 复制（Ctrl+C）
3. 打开公众号后台编辑器
4. 粘贴（Ctrl+V）
5. 填写标题
6. 设置封面图
7. 发布

# 拆书流程

## 流程

### Step 0：准备
- 确认 `参考书/` 目录存在，否则创建
- 检查是否有散落在 `参考书/` 根目录的 txt 文件（不在子文件夹内的）
- 若用户提供了书名但 txt 在根目录，自动定位

### Step 1：建文件夹 + 搬文件
- 在 `参考书/` 下以书名创建文件夹（如 `参考书/六爻/`）
- 把原 txt 文件移入该文件夹
- 路径清洗：文件名中的特殊字符保留原样

### Step 2：调 deconstruction-agent 深度拆书
- 使用 `Agent` 工具调用 `webnovel-writer:deconstruction-agent`
- 参数：reference_text_path = txt 路径, analysis_mode = deep
- 要求返回包含逐章数据的 JSON，每个章节至少包含：章号、字数、开篇钩子类型、章末钩子类型、核心事件/节拍、节奏类型（开场钩子/展开/小高潮/过渡/蓄势/爆发/收束）
- 保留原有的 `init_reference_research` 顶层结构，增加 `chapters` 数组

### Step 3：生成输出文件

#### 3.1 拆书报告 `{书名}-拆书报告.md`
根据 deconstruction-agent 返回的 JSON 编写中文可读报告，格式固定：
- 基本信息（作者/题材/字数/章数）
- 卷结构表
- 剧情主线（200-300字）
- 核心特色（金手指、感情线、叙事结构）
- 可借鉴模式（至少3条）
- 不可复制项
- 差异化要求

#### 3.2 拆书数据 `{书名}-拆书数据.json`
原样保存 deconstruction-agent 返回的 JSON

#### 3.3 章节规划 `{书名}-章节规划.md`
根据 deconstruction-agent 返回的逐章数据生成，格式固定：

```
# {书名} 章节规划

## 第1卷：{卷名}（第1章-第N章）

### 第1章 {章名}
- **字数**：2800
- **开篇钩子**：主角被宗门当众除名 + 全场羞辱
- **章末钩子**：神秘人从人群中注视主角离开
- **节奏类型**：开场钩子
- **核心节拍**：被逐出师门 → 当众撕毁身份文牒 → 同门嘲讽 → 离开山门 → 神秘人尾随
- **信息释放**：世界观基石（宗门等级/诡异降临背景）、主角初始状态（启灵境/弃子身份）
- **情绪起点→终点**：压抑 → 隐怒

### 第2章 {章名}
...
```

每章字段说明：
| 字段 | 内容 |
|------|------|
| 字数 | 该章实际字数 |
| 开篇钩子 | 前300字内用什么拉住读者（冲突/悬念/信息差/金句/场景） |
| 章末钩子 | 最后一节如何制造"继续读"的动力 |
| 节奏类型 | 开场钩子 / 展开 / 小高潮 / 过渡 / 蓄势 / 爆发 / 收束 |
| 核心节拍 | 该章的情节拍序列（3-6个，用→连接） |
| 信息释放 | 该章释放了哪些新信息给读者 |
| 情绪起点→终点 | 章节开头到结尾的情绪走向 |

卷首加一段节奏分析，格式：
```
## 第1卷节奏分析
- 总字数：XXX
- 章节数：XX
- 平均每章字数：XXXX
- 节奏分布：开场钩子 X 章 / 展开 X 章 / 小高潮 X 章 / 过渡 X 章 / 蓄势 X 章 / 爆发 X 章 / 收束 X 章
- 前10章节奏公式：钩→展→展→钩→蓄→爆→收→过→蓄→爆
- 快慢比：快节奏 X 章 / 慢节奏 X 章
```

#### 3.4 情绪曲线 `{书名}-情绪曲线.md`
根据逐章摘要绘制全书情绪概览：
- 按卷分段，每卷标注整体情绪走向（上行/下行/转折/平稳）
- 标注关键情绪锚点：开篇钩子、催泪点、爽点爆发、虐点、收尾情绪
- 可选：用 ASCII 简单图表示意

#### 3.5 节奏统计 `{书名}-节奏统计.md`
从卷结构和逐章数据估算节奏数据：
- 总字数、平均每章字数
- 卷间字数分布
- 卷末高潮章节密度
- 节奏类型分布（每种节奏类型占多少章）
- 前10章节奏公式
- 类型标签（快节奏/慢热/均衡）

### Step 4：验证
- 文件夹存在且包含：原始 txt + 5 个生成文件
- 拆书报告含全部必需字段
- 章节规划含全部必需字段

## 输出清单

`参考书/{书名}/`
├── {原书}.txt
├── {书名}-拆书报告.md
├── {书名}-拆书数据.json
├── {书名}-章节规划.md
├── {书名}-情绪曲线.md
└── {书名}-节奏统计.md

## 调用方式

```
/webnovel-deconstruct 六爻
```

若 `参考书/` 根目录有多个 txt 且未指定书名，列出可选并让用户选择。

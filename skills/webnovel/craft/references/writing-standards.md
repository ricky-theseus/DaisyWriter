# 写作工艺标准手册

> 跨书通用强约束。每条规则含来源、阈值和阻断理由。书项目自身的 `设定集/行文约束集.md` 可追加本书特殊约束，但不得放宽本文标准。

---

## 一、句长控制

### 来源

对六爻（priest）正文采样 5 万字，计算结果：

| 指标 | 六爻 | 我们的标准 | 为何放宽 |
|------|------|:---:|------|
| 平均句长 | 45.6 字 | 30-55 字 | priest 是顶尖作者，我们的 agent 达不到，设可达区间 |
| 极短句占比(≤8字) | 6% | ≤15% | 留白同样有效但容易失控，设硬上限 |
| 长句占比(>30字) | 高 | ≥30% | 长句=描写密度+情绪层次，低则行文偏白 |

### 阻断逻辑

| 情况 | 严重度 | 理由 |
|------|:---:|------|
| 平均句长 <25 | **blocking** | 过短=白描思维，缺情感层次和氛围铺垫 |
| 平均句长 >55 | **blocking** | 过长=阅读疲劳，句号太少角色行动感不足 |
| 极短句 >15% | **blocking** | 碎句太多=节奏崩坏，读者难以沉浸 |
| 平局句长 <30 | warning | 偏短但不致命，部分章节可能合理（战斗） |
| 平均句长 >50 | warning | 偏长但不致命，可留意段落呼吸 |
| 长句 <30% | warning | 偏白但不阻断，可能是快速过渡段 |

### 多章节趋势检查（持久审查 agent 职责）

- 句长均值随卷推进持续下降 → **叙事疲劳预警**
- 句长均值随卷推进持续上升 → **描写冗余预警**
- 相邻两章句长跳变 >15 字 → 风格断裂预警

## 二、句式与修辞

| 规则 | 阈值 | 严重度 | 理由 |
|------|:---:|:---:|------|
| "不是A是B" / "非A而B" 折反句式 | ≤3/章 | >3 → **blocking** | AI 惯用句式，超过 3 次即机械感明显 |
| 身体部位拟人（"眼睛说""手指在颤抖"） | ≤4/章 | >4 → **blocking** | 非必要拟人=AI 凑字数记号 |
| 段落末尾抽象总结/哲理收束 | ≤2/章 | >2 → **blocking** | 网文读者看故事不看哲学，超过 2 次变成说教 |
| 连续句首同主语（"他/她的[部位][动词]"开头） | ≤2句 | >2 → warning | 连续三句以上同主语=句式单调 |

## 三、感官优先级

**触觉 > 听觉 > 视觉**（反直觉约束，目的是激活读者身体感受）

| 情况 | 检查方式 | 严重度 |
|------|------|:---:|
| 连续 500 字纯视觉描写 | 扫描脚本计数 | warning |
| 连续 1000 字纯视觉描写 | 扫描脚本计数 | **blocking** |
| 全章零触觉描写 | reviewer 判断 | warning |

## 四、情绪写作红牌

| 规则 | 说明 | 严重度 |
|------|------|:---:|
| 禁止情绪标签词 | "感到愤怒""充满恐惧""十分悲伤"等直接贴标签 | 出现 → **blocking** |
| 禁止副词修饰情绪动词 | "微微颤抖""缓缓流泪""轻轻叹息" | 出现 → **blocking** |
| 正文:心理分析 ≤ 1:4 | 每 1 句心理分析至少配 4 句外部行为/对话/环境 | reviewer 判断，失衡 → **blocking** |

正确示范：用身体反应、动作犹豫、环境映射传达情绪，而非命名情绪。

## 五、段落节奏

| 规则 | 阈值 | 严重度 |
|------|:---:|:---:|
| 段落≤7句 | 超过 → warning | 大段落降低网文可读性 |
| 静态段落（纯描写/心理，无对话无动作）占比 | ≤50% | **blocking** |
| 连续静态段落 | ≤3段 | warning |
| 情绪单调段落（连续情绪同质）| reviewer 判断 | → warning |

## 六、专家审查五维度（豆包反馈批量化）

每章 reviewer 必须逐项评估。**源自豆包对《未命名者》前 10 章的批评，已抽象为泛用标准。**

### 6.1 信息密度

- 每 500 字必须有新信息或剧情推进
- 以下算"新信息"：角色决策变化、环境新细节、世界观揭示、关系状态变化、伏笔埋设
- 以下不算：重复描写已知道的环境、角色内心反复纠结同一问题无进展
- reviewer 抽样判定，连续 3 个 500 字块无推进 → **blocking**

### 6.2 主角主动性

- 主角（或 ensemble 核心角色之一）必须在本章做出至少一次影响剧情的主动选择
- 主动选择判据：①有至少两个可选项 ②选择带来明确后果 ③不做选择剧情会不同
- 以下不算：被动反应（被攻击后反击）、被迫服从（命令/威胁下的行动）、旁观
- 全章无主动选择 → **blocking**
- 群像文：至少一个核心角色做出主动选择即可通过

### 6.3 情绪张弛

- 必须有至少一次"压抑→释放"的情绪转折
- 可以是微小的：期待→失落、紧张→松弛、孤独→被注意
- 全章情绪单调整（一直紧张或一直平缓）→ **warning**
- 连续 3 章无情绪转折 → **blocking**（持久审查 agent 检查）

### 6.4 主支线权重

- 主线剧情篇幅必须大于支线
- 支线（角色日常、铺垫、闲笔）总计不超过总字数 40%
- 支线在章节前 1/3 出现 → OK（先铺垫再主线）
- 支线在章节后 1/3 出现 → **warning**（削弱章末钩子）
- reviewer 参照章纲 `CBN` 节点判定主线/支线归属

### 6.5 节奏推进

- 检查链条：开篇→事件发生→角色应对→后果→选择→新方向
- 链条断在"事件发生"（开篇后无推动）→ **blocking**
- 链条断在"后果"（事件后直接跳转无反应时间）→ warning
- 链条断在"选择"（角色被动接受后果无决策）→ 合并检查 6.2 主角主动性

## 七、两阶段审查架构

### 设计理由

量化扫描（`prose_scanner.py`）负责 LLM 不擅长的精确计数任务（句式计数、句长统计、词库扫描），让 reviewer agent 专注于品味判断（情绪温度、节奏感、文学质量）。

### 执行顺序

1. **阶段一**：`prose_scanner.py` → 产出 JSON，标记 blocking/warning 项
2. **阶段二**：reviewer agent → 审查五维度（品控不对机械内容作重复判断）
3. blocking 项定点修复 → 脚本回归 → 不重新跑 reviewer（除非改动涉及品味层面）
4. 修复后仍 blocking → 交用户裁决

### blocking 不可自动修复的情况

- 句长分布极端（<20 或 >70，修复=重写全章）
- 情绪标签词遍布全章（修复=重新表达所有情绪段）
- 结构性问题（缺 CBN 节点、时间线断裂）
- 以上情况标记 failed 并暂停，不做静默降级

## 八、行文约束集模板

每个书项目应在 `设定集/行文约束集.md` 建立本书特殊约束。模板结构：

```markdown
# 行文约束集

> 本项目写作约束 + 审查标准。
> 本文约束叠加于 webnovel-craft 通用标准之上，不得放宽通用标准。
> 违规则写 issue。blocking 判定：触发硬红线=blocking，偏离偏好/有可替代方案=warning。

## 一、句式与修辞（本书）
## 二、段落节奏（本书）
## 三、感官优先级（本书）
## 四、情绪温度（本书）
## 五、跨章一致性（本书）
## 六、本书特约（特殊设定、禁忌、世界观独有约束）

```

通用约束（句长、反AI句式、感官优先级基线）已在本文第一条定义，书项目无需重复；本书模板仅填写"超出通用标准之外"的约束。

## 九、Anti-AI 7-Gate 检测

七道门禁用于系统性清除 AI 写作痕迹。检测流程：扫描 → 分级 → 逐门清除 → 复扫验证。

### Gate A：禁用词替换

对照禁用词表检测，hit 即改。替换规则：禁用词 → 具体动作/细节描写，不换成另一个形容词。白名单机制：项目 `.deslop-whitelist` 中的词汇豁免检测。

### Gate B：句式去套路

| 句式 | 问题 | 替代方案 |
|------|------|----------|
| 否定铺垫后接肯定翻转 | 中文 AI 最毒句式之一 | 直接写后项，或改成动作/细节呈现 |
| "...，带着..." | 万能状语，AI 最爱 | 用独立短句或动作描写 |
| "声音不大，却带着……" | AI 最爱声音描写 | 直接写声音特征或动作 |
| "仿佛/犹如/宛如/如同" | 文言腔过重 | 口语化表达或白描 |
| "不容置疑/显而易见" | 书面化判断词 | 用具体事实说话 |
| 连续三个以上排比 | 过于工整 | 保留 1-2 个，删掉其余 |

修饰词清扫：物品/人物前面的形容词、定语、副词、指示代词、量词，多余即删。

### Gate C：心理描写外化

直接陈述情绪 → 用行为展示：
- "他很紧张" → "他的手在抖"
- "她很愤怒" → "她一把掀翻了桌子"
- 同一信息/动作/情绪在相邻段重复 → 合并去重，保留最能推动情绪或剧情的细节

### Gate D：节奏打碎

- 打断连续排比句（保留1-2个，删掉其余）
- 长句拆短句，偶尔用不完整句
- 段落长短交错，不每段都 3-5 行
- 标点节奏跟语气走：保留有功能的 `？`/少量 `！`，不保留 `……`/`——`

### Gate E：对话去腔调

- 加入口语化表达（"嗯""哦""行吧"）
- 适当打断对话，用动作穿插对话
- 删掉解释性对话（角色不会把自己的动机说清楚）
- 角色说话有差异，不千人一面

### Gate F：结尾去升华

- 删掉总结性语句，用动作/场景收尾
- 结尾有"他知道...""这一刻..."基本可以删

### Gate G：去解释腔/上帝感/安排感

最难察觉的 AI 味。叙述者跳出角色当下去解释、剧透、总结：
- 删解释因果：「之所以…是因为」「原来…」「这意味着」
- 删上帝视角剧透：「她不知道的是」「殊不知」「仿佛预示着」
- 删替读者定性：「演得真好」「他就是这样薄情」
- 删隐蔽软评判：评判性副词、剧透点破、定性比喻

### AI 味分级与处理策略

| 程度 | 禁用词密度 | 特征 | 处理 |
|------|-----------|------|------|
| 轻度 | ≤5/千字 | 少量禁用词，偶有书面腔 | 只过 Gate A+B |
| 中度 | 6-15/千字 | 多处禁用词+句式套路+心理描写抽象 | 过 Gate A+B+C+D+G |
| 重度 | >15/千字 | 全文明显，节奏/对话/结尾/解释腔均有问题 | 完整 7 Gate + 重点段落重写 |

综合判定：取六项指标（禁用词密度、连续排比段数、心理词占比、对话标签密度、平均段落句数、重复描写密度）的最高档位。

### 自然表达替换参考

- "深吸一口气" → "胸口起伏了一下" / 直接删掉
- "眼中闪过一丝..." → "他垂下眼" / "眯起眼"
- "嘴角勾起一抹..." → "笑了一下，没到眼底" / "乐了"
- "仿佛..." → "像..." / 直接白描
- "不禁..." → 直接写动作
- "缓缓开口" → "说" / 用动作引出对话

## 十、中文文学技法

### 白描（Plain Sketch）

用最少的笔墨精确捕捉本质。名词和动词为主，形容词能砍就砍。

反例：璀璨的星空下，瑰丽的晚霞映照着静谧的湖面。
正例：湖面映着天光，水波不兴。远处有人点了一盏灯。

操作要点：
- 一句话最多一个修饰语
- 动作代替形容："他老了"不如"他扶着墙站起来"
- 写完一段后，删掉所有形容词再读一遍，如果意思没变，那些形容词就是多余的

### 留白（Leaving Blank Space）

不说的部分比说的部分更有力量。在关键场景的解答前收笔，让读者自己填充。

操作要点：
- 对话场景：在沉默的那一刻收尾，不要写出回答
- 情感场景：写动作，不写感受。"她把信折好放回信封"比"她心如刀绞"有力
- 转折场景：写到转折发生的那一瞬间停止，下一章再接
- 高潮场景：最高潮处戛然而止，不要写角色的感想和总结

### 意象营造（Imagery）

用一个反复出现的具象物品承载抽象情感。它是情感的速记符号。

操作要点：
- 选取与人物或主题相关的具象物品（一朵枯花、一只停摆的钟）
- 前期自然出现，不刻意强调
- 中期变化重复（花从新鲜到枯萎）
- 关键时刻集中爆发，释放全部累积的情感重量
- 一本书里意象不宜超过三个，否则变成密码本

### 草蛇灰线（Subtle Foreshadowing）

在看似随意的细节中埋入线索。读者第一次读到不会注意，重读时才会拍案。

操作要点：
- 埋伏笔时伪装成无关紧要的闲笔
- 揭晓时不要大张旗鼓，让读者自己发现那个联系
- 间隔要足够远（至少隔五章以上）
- 伏笔清单必须记录，否则写着写着就忘了

### 蒙太奇剪辑（Montage Editing）

用锐利的场景跳跃制造意义。不做平滑过渡，让并列本身讲故事。

操作要点：
- 两个场景之间直接切换，不用"与此同时""另一边"等过渡语
- 用空行或分隔符标记跳跃
- 选择有强烈对比或呼应的两组画面
- 对比产生第三层含义：A+B=C（C不在文本里，在读者脑子里）

## 十一、质量评分系统（80分制）

每章交付前对照打分。>60 分可交付，>70 分优秀。

| 维度 | 评分 | 检查要点 |
|------|:----:|---------|
| 开头吸引力 | /10 | 前 3 段抓住读者？是否在第一段建立冲突或紧张？ |
| 情节推进 | /10 | 本章推进主线？有核心事件（删除它会影响理解）？ |
| 人物塑造 | /10 | 人物行为一致且有深度？对话能区分不同角色？ |
| 对话质量 | /10 | 对话自然且推动情节？有潜台词，不是所有话都直说？ |
| 悬念设置 | /10 | 结尾钩子让读者想看下一章？悬念强度与故事位置匹配？ |
| 节奏控制 | /10 | 张弛有度？信息密度有高低交替？段落长短变化？ |
| 展示而非讲述 | /10 | 用行动/对话而非直接陈述情绪？动词优于形容词？ |
| 语言质量 | /10 | 无 AI 痕迹，用词精确，"的"字不超 2 个/句？ |

评分规则：
- 每一项对照检查清单打分，不凭感觉
- 如果某项低于 6 分，该维度必须修改后重新评分
- 总分低于 60 分：整章需要大改
- 总分 60-70 分：基本可交付，打磨细节

## 十二、AI 高频词汇黑名单

以下词汇出现即改：

### 书面连接词
"此外""然而""值得注意的是""需要强调的是""不可忽视""与此同时""这意味着"

### 抽象动词
"彰显""诠释""赋能""映射""折射""铭记""启程"

### AI 情绪套话
"不禁""油然而生""心潮澎湃""百感交集""五味杂陈"

### 万能过渡词
"这一刻""仿佛""宛如""终究""终于""忽然"

### 叙述性判断
"不容置疑""显而易见""毫无疑问""众所周"

### 章末总结体
"这一切都说明...""他终于明白...""新的篇章开始了..."

### 四字成语堆砌
- 连续使用两个以上四字成语就属于堆砌
- 搜索文中所有四字成语，把连续出现的拆开或替换为口语表达

### "的"字密度规则
- 一句话里"的"不超过两个
- 绝对不能出现连续的"的的"
- 修饰结构能简化就简化："他的红色的漂亮的大衣"→"他那件红色大衣"

### 模糊词（删除或替换）
"一些""某种""大概""似乎""有点""稍微"（除非刻意表达不确定性）

# 写作工艺标准手册

> 跨书通用强约束。每条规则含来源、阈值和阻断理由。书项目自身的 `设定集/行文约束集.md` 可追加本书特殊约束，但不得放宽本文标准。

---

## 一、句长控制

### 来源

对六爻（priest）正文采样 5 万字，计算结果：

| 指标 | 六爻 | 我们的标准 | 为何放宽 |
|------|------|:---:|------|
| 平均句长 | 45.6 字 | 30-55 字 | priest 是顶尖作者，我们的 agent 达不到，设可达区间 |
| 极短句占比(≤8字) | 6% | ≤15% | 留白同样有效但容易失控，设硬上限 |
| 长句占比(>30字) | 高 | ≥30% | 长句=描写密度+情绪层次，低则行文偏白 |

### 阻断逻辑

| 情况 | 严重度 | 理由 |
|------|:---:|------|
| 平均句长 <25 | **blocking** | 过短=白描思维，缺情感层次和氛围铺垫 |
| 平均句长 >55 | **blocking** | 过长=阅读疲劳，句号太少角色行动感不足 |
| 极短句 >15% | **blocking** | 碎句太多=节奏崩坏，读者难以沉浸 |
| 平局句长 <30 | warning | 偏短但不致命，部分章节可能合理（战斗） |
| 平均句长 >50 | warning | 偏长但不致命，可留意段落呼吸 |
| 长句 <30% | warning | 偏白但不阻断，可能是快速过渡段 |

### 多章节趋势检查（持久审查 agent 职责）

- 句长均值随卷推进持续下降 → **叙事疲劳预警**
- 句长均值随卷推进持续上升 → **描写冗余预警**
- 相邻两章句长跳变 >15 字 → 风格断裂预警

## 二、句式与修辞

| 规则 | 阈值 | 严重度 | 理由 |
|------|:---:|:---:|------|
| "不是A是B" / "非A而B" 折反句式 | ≤3/章 | >3 → **blocking** | AI 惯用句式，超过 3 次即机械感明显 |
| 身体部位拟人（"眼睛说""手指在颤抖"） | ≤4/章 | >4 → **blocking** | 非必要拟人=AI 凑字数记号 |
| 段落末尾抽象总结/哲理收束 | ≤2/章 | >2 → **blocking** | 网文读者看故事不看哲学，超过 2 次变成说教 |
| 连续句首同主语（"他/她的[部位][动词]"开头） | ≤2句 | >2 → warning | 连续三句以上同主语=句式单调 |

## 三、感官优先级

**触觉 > 听觉 > 视觉**（反直觉约束，目的是激活读者身体感受）

| 情况 | 检查方式 | 严重度 |
|------|------|:---:|
| 连续 500 字纯视觉描写 | 扫描脚本计数 | warning |
| 连续 1000 字纯视觉描写 | 扫描脚本计数 | **blocking** |
| 全章零触觉描写 | reviewer 判断 | warning |

## 四、情绪写作红牌

| 规则 | 说明 | 严重度 |
|------|------|:---:|
| 禁止情绪标签词 | "感到愤怒""充满恐惧""十分悲伤"等直接贴标签 | 出现 → **blocking** |
| 禁止副词修饰情绪动词 | "微微颤抖""缓缓流泪""轻轻叹息" | 出现 → **blocking** |
| 正文:心理分析 ≤ 1:4 | 每 1 句心理分析至少配 4 句外部行为/对话/环境 | reviewer 判断，失衡 → **blocking** |

正确示范：用身体反应、动作犹豫、环境映射传达情绪，而非命名情绪。

## 五、段落节奏

| 规则 | 阈值 | 严重度 |
|------|:---:|:---:|
| 段落≤7句 | 超过 → warning | 大段落降低网文可读性 |
| 静态段落（纯描写/心理，无对话无动作）占比 | ≤50% | **blocking** |
| 连续静态段落 | ≤3段 | warning |
| 情绪单调段落（连续情绪同质）| reviewer 判断 | → warning |

## 六、专家审查五维度（豆包反馈批量化）

每章 reviewer 必须逐项评估。**源自豆包对《未命名者》前 10 章的批评，已抽象为泛用标准。**

### 6.1 信息密度

- 每 500 字必须有新信息或剧情推进
- 以下算"新信息"：角色决策变化、环境新细节、世界观揭示、关系状态变化、伏笔埋设
- 以下不算：重复描写已知道的环境、角色内心反复纠结同一问题无进展
- reviewer 抽样判定，连续 3 个 500 字块无推进 → **blocking**

### 6.2 主角主动性

- 主角（或 ensemble 核心角色之一）必须在本章做出至少一次影响剧情的主动选择
- 主动选择判据：①有至少两个可选项 ②选择带来明确后果 ③不做选择剧情会不同
- 以下不算：被动反应（被攻击后反击）、被迫服从（命令/威胁下的行动）、旁观
- 全章无主动选择 → **blocking**
- 群像文：至少一个核心角色做出主动选择即可通过

### 6.3 情绪张弛

- 必须有至少一次"压抑→释放"的情绪转折
- 可以是微小的：期待→失落、紧张→松弛、孤独→被注意
- 全章情绪单调整（一直紧张或一直平缓）→ **warning**
- 连续 3 章无情绪转折 → **blocking**（持久审查 agent 检查）

### 6.4 主支线权重

- 主线剧情篇幅必须大于支线
- 支线（角色日常、铺垫、闲笔）总计不超过总字数 40%
- 支线在章节前 1/3 出现 → OK（先铺垫再主线）
- 支线在章节后 1/3 出现 → **warning**（削弱章末钩子）
- reviewer 参照章纲 `CBN` 节点判定主线/支线归属

### 6.5 节奏推进

- 检查链条：开篇→事件发生→角色应对→后果→选择→新方向
- 链条断在"事件发生"（开篇后无推动）→ **blocking**
- 链条断在"后果"（事件后直接跳转无反应时间）→ warning
- 链条断在"选择"（角色被动接受后果无决策）→ 合并检查 6.2 主角主动性

## 七、两阶段审查架构

### 设计理由

量化扫描（`prose_scanner.py`）负责 LLM 不擅长的精确计数任务（句式计数、句长统计、词库扫描），让 reviewer agent 专注于品味判断（情绪温度、节奏感、文学质量）。

### 执行顺序

1. **阶段一**：`prose_scanner.py` → 产出 JSON，标记 blocking/warning 项
2. **阶段二**：reviewer agent → 审查五维度（品控不对机械内容作重复判断）
3. blocking 项定点修复 → 脚本回归 → 不重新跑 reviewer（除非改动涉及品味层面）
4. 修复后仍 blocking → 交用户裁决

### blocking 不可自动修复的情况

- 句长分布极端（<20 或 >70，修复=重写全章）
- 情绪标签词遍布全章（修复=重新表达所有情绪段）
- 结构性问题（缺 CBN 节点、时间线断裂）
- 以上情况标记 failed 并暂停，不做静默降级

## 八、行文约束集模板

每个书项目应在 `设定集/行文约束集.md` 建立本书特殊约束。模板结构：

```markdown
# 行文约束集

> 本项目写作约束 + 审查标准。
> 本文约束叠加于 webnovel-craft 通用标准之上，不得放宽通用标准。
> 违规则写 issue。blocking 判定：触发硬红线=blocking，偏离偏好/有可替代方案=warning。

## 一、句式与修辞（本书）
## 二、段落节奏（本书）
## 三、感官优先级（本书）
## 四、情绪温度（本书）
## 五、跨章一致性（本书）
## 六、本书特约（特殊设定、禁忌、世界观独有约束）
## 七、自查清单
```

通用约束（句长、反AI句式、感官优先级基线）已在本文第一条定义，书项目无需重复；本书模板仅填写"超出通用标准之外"的约束。

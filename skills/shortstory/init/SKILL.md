# 短篇初始化 + 盲审迭代

## 目标

在单个session内完成N篇短篇的选题→设定文档→盲审→迭代修改→交付，产出可直接开写的项目文件夹。

## 流程

### Phase 0：定选题（主进程 × 用户）

- 与用户确认：篇数、方向、目标字数、投稿征稿活动
- 参考 `短篇/候选/` 下的现有选题（如有）
- 每篇确定：类型（悬疑/言情/脑洞/古风）、核心设定一句话、太仓元素锚点

### Phase 1：初始化文档（主进程写）

为每篇创建项目文件夹 `短篇/{类型}/{项目名}/`，包含：

| 文件 | 内容 | 行数参考 |
|------|------|----------|
| `作品信息.md` | 作品名称、金句、简介、标签、主角、人设 | ~30行 |
| `设定.md` | 世界观、核心规则、关键道具/机制、时间线 | ~80行 |
| `角色.md` | 主角/配角/反派，每人：身份、动机、弧光 | ~40行 |
| `章节规划.md` | 章数、每章场景、节奏、伏笔回收表、工具人预案 | ~200行 |
| `工艺约束.md` | 引用 `shortstory-craft` + `webnovel-craft`，叠加本篇特殊约束 | ~20行 |
| `正文.md` | 空文件，占位 | 1行 |

约束：
- 设定必须100%自洽，无互斥
- 章纲必须包含：每章字数、开篇钩子、场景细分、结尾收束方式
- 伏笔回收表必须注明出现章和回收章
- 工具人预案：警方为何不介入、文物所为何不封锁等

### Phase 2：盲审（子agent × 无记忆）

对每篇的项目文档，创建一个**独立子agent**做设计预审。

子agent prompt 模板：

```
你是一个毒舌但公平的短篇编辑（类似知乎盐选制作人），没有看过这个项目的任何先前提案。
请审核以下短篇项目的设定文档和章节规划，输出问题清单。

审核维度（按优先级）：
1. 逻辑自洽性：设定有无互斥、时间线有无矛盾、规则是否一致
2. 角色弧光：每个角色的动机是否成立、转变有无铺垫
3. 节奏控制：章与章之间信息量是否均衡、高潮位置是否合理
4. 伏笔设计：伏笔有无回收路径、反转是否可预期
5. 工具人漏洞：配角行为是否符合身份、有无"为什么警察不查"式漏洞
6. 番茄/盐选适配：开篇钩子是否在前2000字内、情绪是否足以留住读者

输出格式：
- 【阻断级】问题：必须改，不改不能开写
- 【警告级】问题：建议改，影响质量上限
- 【建议级】问题：改了更好，不强求
```

### Phase 3：迭代修改（主进程）

- 根据子agent的【阻断级】问题逐一修改设定/章纲
- 每改完一轮，重新创建**新的**子agent（仍无记忆）再审
- 迭代直到子agent输出零阻断（允许少量警告和建议）
- 每篇独立迭代，互不干扰

### Phase 4：交付确认

- 完成所有N篇的零阻断迭代后，逐篇展示给用户确认
- 用户确认后，项目文件夹可被新session直接使用

## 子agent管理规则

| 要点 | 规则 |
|------|------|
| 身份 | 纯审核者，无作者记忆 |
| 创建时机 | 每轮迭代新建，不复用旧session |
| 输入 | 仅项目文件夹内容（设定.md + 章节规划.md + 角色.md） |
| 输出 | 分优先级的 issue list |
| 终止条件 | 零阻断 |

## 输出目录结构

```
短篇/{类型}/{项目名}/
├── 作品信息.md
├── 设定.md
├── 角色.md
├── 章节规划.md
├── 工艺约束.md
└── 正文.md    (空，留给写阶段)
```

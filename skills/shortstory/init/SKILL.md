# 中短篇初始化（状态机驱动）

## 与 shortstory-write 的分工

| skill | 负责 |
|-------|------|
| `shortstory-init` | **当前 skill**。选题→设定文档→文件完整性门禁→盲审迭代→产出初始化文件夹 |
| `shortstory-write` | 滚动写。单文件正文.md，字数门禁 + 状态机驱动 |

init 产出的是**设计蓝图**，write 按图施工。init 完成后不碰产出文件夹。

## 状态机

```
        ┌─────────────────────────────────────────────┐
        │              初始状态机                      │
        │                                             │
        │  ┌──────────┐  文件完整性   ┌──────────┐    │
        │  │ drafting │ ──────────→ │ in_review │    │
        │  │ 写设定文件 │  validate    │  待盲审   │    │
        │  └────┬─────┘              └────┬──────┘    │
        │       │ ▲                       │           │
        │    文件│ ║                   盲审│           │
        │    不完整 ║                   查出│阻断       │
        │       │ ║                       ▼           │
        │       │ ║                 ┌──────────┐      │
        │       │ ║                 │ blocking │      │
        │       │ ║                 │  有阻断   │      │
        │       │ ║                 └────┬─────┘      │
        │       └╩═══════════════════════╝            │
        │                修完回写                     │
        │                                            │
        │       盲审零阻断                            │
        │   in_review ──────────→ passed ──→ done    │
        └────────────────────────────────────────────┘
```

## 状态定义

| 状态 | 含义 | 触发 |
|------|------|------|
| `drafting` | 正在创建/修改设定文件 | 初始状态 |
| `in_review` | 文件完整，等待盲审 | validate_init.py 文件完整性通过后自动转 |
| `blocking` | 盲审查出阻断级问题 | agent 根据盲审结果手动设 |
| `passed` | 零阻断，初始化完成 | agent 根据盲审结果（零阻断）手动设 |

## 流程

### 0. 定选题（用户交互）

- 确认：篇数、方向、目标字数、征稿活动
- 参考 `短篇/候选/` 现有选题

### 1. 写设定文件（drafting）

为每篇创建 `短篇/{类型}/{项目名}/`，写 6 个文件：

| 文件 | 内容 | 要求 |
|------|------|------|
| `作品信息.md` | 名称、金句、简介、标签、主角人设 | — |
| `设定.md` | 世界观、核心规则（逐条编号）、关键道具、时间线 | ≥200 字 |
| `角色.md` | 每人：身份、动机、弧光，关系 ASCII 图 | — |
| `章节规划.md` | 章数、每章场景/钩子/收束、伏笔回收表、工具人预案 | 必须含 `第N章` 条目 |
| `工艺约束.md` | 继承 craft，叠加本篇特殊约束 | — |
| `正文.md` | 空文件占位 | — |

约束：
- 设定 100% 自洽
- 伏笔回收表注明出现章和回收章
- 工具人预案覆盖"警方不介入"类逻辑漏洞

### 2. 文件完整性门禁

```bash
python <skill_dir>/validate_init.py 短篇/{类型}/{项目名}/
```

脚本检查：
- 6 个文件全部存在且非空
- `章节规划.md` 至少含一条 `第N章` 条目
- `设定.md` ≥ 200 字

**输出（通过）：** `{"ok": true}` → 状态自动从 `drafting` 转为 `in_review`

**输出（不通过）：** `{"ok": false, "errors": [...]}` → 状态保持 `drafting`，必须补全

### 3. 盲审（in_review → blocking / passed）

文件完整性通过后（状态为 `in_review`），新建无记忆子 agent：

**输入：** 设定.md / 角色.md / 章节规划.md / 作品信息.md / 工艺约束.md

**审核维度（按优先级）：**
1. 逻辑自洽：设定互斥、时间线矛盾
2. 角色弧光：动机成立、转变有铺垫
3. 节奏控制：章间信息量均衡
4. 伏笔设计：回收路径
5. 工具人漏洞
6. 盐选适配：开篇钩子

**输出：** 【阻断级】/ 【警告级】/ 【建议级】

**根据结果更新状态文件：**

- **有阻断级** → `status` 改为 `blocking`，阻断条目写入 `review_blocking`
  - 回到步骤 1 修改文件
  - 改完 → 步骤 2 文件门禁 → 步骤 3 盲审 → 循环直到零阻断

- **零阻断** → `status` 改为 `passed`

### 4. 交付确认

- 零阻断后展示给用户确认
- 项目文件夹可直接被 `shortstory-write` 使用

## 校验脚本

```
python <skill_dir>/validate_init.py 短篇/{类型}/{项目名}/
```

脚本只修改 `drafting → in_review` 自动转移，其他状态变更由 agent 手动写入。

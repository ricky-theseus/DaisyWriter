# 拆解短篇故事

## 流程

### Step 0：准备
- 用户提供 md/txt 文件路径，或直接粘贴原文
- 确认归类到哪个类型（言情/悬疑/脑洞/古风/其他）

### Step 1：建目录 + 搬文件
- 在 `短篇/参考书/{类型}/` 下以文章名创建文件夹
- 把原文存为 `原文.md`

### Step 2：拆解分析
逐维度拆解原文，输出结构化数据：

| 维度 | 检查点 |
|------|--------|
| 开场钩子 | 前 500 字用什么吸引读者（痛点/悬念/金句/场景） |
| 结构骨架 | 开场→展开→转折→高潮→收束，各段字数占比 |
| 悬念设计 | 埋在哪里，如何释放，有没有"钩子链" |
| 节奏控制 | 对话/叙述/描写的交替频率，快慢段落分布 |
| 信息密度 | 每 500 字是否有新信息推进（角色/关系/世界观/冲突） |
| 情绪曲线 | 从开场到结尾的情绪走势图，关键情绪锚点 |
| 人物塑造 | 短篇篇幅内如何快速建立人物形象（标签/对话/行动） |
| 结尾质量 | twist 类型，情绪收束方式，是否留回味 |
| 可借模式 | 可复用的结构套路/叙事手法/句式节奏 |

### Step 3：生成文件
- `原文.md` — 清洗后的原文
- `拆解报告.md` — 中文可读报告
- `拆解数据.json` — 结构化数据

## 输出清单

```
短篇/参考书/{类型}/{文章}/
├── 原文.md
├── 拆解报告.md
└── 拆解数据.json
```
